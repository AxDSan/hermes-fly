## Goal

Deploy a Telegram bot to Fly.io, get an intelligent reply — with **zero
manual post-deploy steps**. v0.1.10 closes four gaps to make this the
default experience. Success: the bot responds naturally to user input:

> "Hey! I'm here and ready to go. What can I help you with?"

### Friction Point #1 Evidence: OpenRouter 401 Error

After running `hermes-fly doctor`, the deployment showed four failures. Later, when sending the bot a message:

**```
[07/03/2026 17:52] test-test-fly-hermes-101: ⚠️ Error code: 401 - {'error': {'message': 'User not found.', 'code': 401}}
```**

The 401 error from OpenRouter indicates the API key was never validated during deploy—it was accepted silently despite being invalid, requiring manual debugging after the bot went live.

### Friction Point #2 Evidence: Pairing Prompt

**```
[07/03/2026 17:51] test-test-fly-hermes-101: Hi~ I don't recognize you yet!

Here's your pairing code: EZTRVCTC

Ask the bot owner to run:
hermes pairing approve telegram EZTRVCTC
```**

Despite the user configuring `TELEGRAM_ALLOWED_USERS=alexfazio` during setup, the bot still required SSH + manual code approval instead of auto-approving the configured user.

### Friction Point #3 Evidence: Doctor False Negatives

**```
[PASS] app: App 'test-test-67' found
[FAIL] machine: Machine not running. Start with: fly machine start -a test-test-67
[PASS] volumes: Volumes attached
[PASS] secrets: Required secrets are set

[PASS] api: LLM API is reachable

The `[FAIL] gateway` check used an HTTP probe (expecting the bot to listen on port 8080), which doesn't apply to Telegram polling bots. This created false negatives when the bot was actually operational.

### Friction Point #4 Evidence: No Fleet Management

The session showed no `hermes-fly list` command and no clean interactive destroy flow — users had no way to list deployed agents or cleanly shut them down.

## Design Clarifications

**Polling vs Webhook:** v0.1.10 uses polling (not webhook-based updates) for Telegram integration. This choice prioritizes simplicity for solo developers managing 1-5 agents, avoiding webhook infrastructure complexity. Webhook support is a future enhancement (v0.1.11+).

**Credential Validation Timing:** Deploy-time validation (Slices 1-2) is sufficient for v0.1.10. Runtime token health checks (e.g., periodic getMe calls) are deferred to v0.1.11+ per operational maturity roadmap.

**Token Cooldown After Destroy:** Telegram's `logOut` endpoint requires a 10-minute cooldown before token reuse. This is documented in CLI output (Slice 7) and deploy summaries; no hard CLI guard is enforced, trusting solo developers to respect the warning.

**Fleet Persistence:** YAML-based state (configs, deploy metadata) is appropriate for individual developers managing <5 agents. Scaling guidance: this approach works well up to ~50 agents; larger fleets should consider a structured state backend (v0.1.12+).

### Step 1.2 — GREEN: `lib/messaging.sh`

**Add new function `messaging_validate_telegram_token_api`** (after line 29):

```bash
# Call Telegram getMe to validate token and get bot identity.
# Sets DEPLOY_TELEGRAM_BOT_USERNAME and DEPLOY_TELEGRAM_BOT_NAME on success.
messaging_validate_telegram_token_api() {
  local token="$1"
  local response
  response="$(curl -sf --max-time 10 "https://api.telegram.org/bot${token}/getMe" 2>&1)" || return 1
  local username
  username="$(printf '%s' "$response" | sed -n 's/.*"username"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p' | head -1)"
  [[ -z "$username" ]] && return 1
  DEPLOY_TELEGRAM_BOT_USERNAME="$username"
  DEPLOY_TELEGRAM_BOT_NAME="$(printf '%s' "$response" | sed -n 's/.*"first_name"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p' | head -1)"
  export DEPLOY_TELEGRAM_BOT_USERNAME DEPLOY_TELEGRAM_BOT_NAME
  return 0
}
```

```bash
messaging_setup_telegram() {
  printf '\n--- Telegram Bot Setup ---\n' >&2
  printf 'To create a Telegram bot:\n' >&2
  printf '  1. Open Telegram and search for @BotFather\n' >&2
  printf '  2. Send /newbot and follow the prompts\n' >&2
  printf '  3. Copy the bot token provided\n' >&2
  printf '\n' >&2

  local token
  while true; do
    ui_ask_secret 'Bot token:' token
    if ! messaging_validate_telegram_token "$token"; then
      printf 'Error: token format invalid. Expected: 123456789:ABCdef...\n' >&2
      continue
    fi
    printf 'Verifying token with Telegram...\n' >&2
    if ! messaging_validate_telegram_token_api "$token"; then
      printf 'Error: Telegram rejected this token. Check it and try again.\n' >&2
      continue
    fi
    printf 'Found bot: @%s (%s)\n' "$DEPLOY_TELEGRAM_BOT_USERNAME" "$DEPLOY_TELEGRAM_BOT_NAME" >&2
    if ui_confirm "Continue with this bot?"; then
      break
    fi
  done

  printf '\nTo find your Telegram user ID:\n' >&2
  printf '  1. Message @userinfobot on Telegram\n' >&2
  printf '  2. It replies with your numeric user ID (e.g., 123456789)\n' >&2
  printf 'Only these IDs can interact with the bot.\n' >&2
  printf 'Leave blank to allow all users.\n' >&2

  local users
  while true; do
    printf 'User IDs (comma-separated, or blank for all): ' >&2
    IFS= read -r users
    if messaging_validate_user_ids "$users"; then
      break
    fi
    printf 'Error: user IDs must be numeric (e.g., 123456789). Use @userinfobot to find yours.\n' >&2
  done

  DEPLOY_TELEGRAM_BOT_TOKEN="$token"
  DEPLOY_TELEGRAM_ALLOWED_USERS="$users"
  export DEPLOY_TELEGRAM_BOT_TOKEN DEPLOY_TELEGRAM_ALLOWED_USERS
  return 0
}

```bash
# Validate OpenRouter API key via /auth/key endpoint.
# Warns if free tier with no credit. Returns 1 on invalid key.
deploy_validate_openrouter_key() {
  local api_key="$1"
  local response
  response="$(curl -sf --max-time 10 "https://openrouter.ai/api/v1/key" \
    -H "Authorization: Bearer ${api_key}" 2>&1)" || return 1
  printf '%s' "$response" | grep -q '"error"' && return 1

  local is_free_tier
  is_free_tier="$(printf '%s' "$response" | sed -n 's/.*"is_free_tier"[[:space:]]*:[[:space:]]*\(true\|false\).*/\1/p' | head -1)"
  local usage
  usage="$(printf '%s' "$response" | sed -n 's/.*"usage"[[:space:]]*:[[:space:]]*\([0-9][0-9]*\).*/\1/p' | head -1)"

  if [[ "$is_free_tier" == "true" ]] && [[ "${usage:-0}" == "0" ]]; then
    printf 'Warning: OpenRouter account is on free tier with no usage. Add credits at https://openrouter.ai/credits\n' >&2
  fi
  return 0
}
```

Replace `doctor_check_gateway_health`:

```bash
doctor_check_gateway_health() {
  local app_name="$1"
  # Check if Telegram bot token is configured
  local secrets_list
  secrets_list="$(fly secrets list --app "$app_name" 2>/dev/null || echo "")"
  if printf '%s' "$secrets_list" | grep -q "TELEGRAM_BOT_TOKEN"; then
    # Use Telegram getMe via the machine's environment (token is in machine env)
    if fly ssh console --app "$app_name" -C \
       'curl -sf --max-time 10 "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/getMe" >/dev/null 2>&1' \
       2>/dev/null; then
      return 0
    fi
    return 1
  fi
  # Fallback: HTTP probe for webhook-based deploys
  local url="https://${app_name}.fly.dev"
  if curl -sf --max-time 10 "$url" >/dev/null 2>&1; then
    return 0
  fi
  return 1
}
```

  printf '  ┌──────────────────────┬────────┬──────────┬─────────┐\n'

    deploy_yaml="${HERMES_FLY_CONFIG_DIR:-$HOME/.hermes-fly}/deploys/${app}.yaml"

Add `destroy_telegram_logout` function:

```bash
# Call Telegram logOut and deleteWebhook for a deployed app.
# Reads bot token via fly ssh console. Fail-open with manual instructions.
# Critical: prints 10-minute cooldown warning to prevent token reuse errors.
destroy_telegram_logout() {
  local app_name="$1"
  printf 'Disconnecting Telegram bot...\n' >&2

  # Run logOut and deleteWebhook inside the machine (token is in env)
  if fly ssh console --app "$app_name" -C \
     'curl -sf "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/logOut" && curl -sf "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/deleteWebhook?drop_pending_updates=true"' \
     2>/dev/null; then
    printf '✓ Telegram bot disconnected (token invalidated)\n' >&2
    printf '⚠ IMPORTANT: Wait 10 minutes before reusing this bot token with hermes-fly deploy\n' >&2
  else
    printf 'Warning: could not disconnect Telegram bot automatically.\n' >&2
    printf 'Manual steps:\n' >&2
    printf '  1. Open @BotFather on Telegram\n' >&2
    printf '  2. Send /mybots → select bot → Edit Bot → Revoke current token\n' >&2
    printf '  3. Revoke OpenRouter key at: https://openrouter.ai/keys\n' >&2
  fi
}
